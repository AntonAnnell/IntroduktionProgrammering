Inlämningsuppgift programmering Del 1


Objekt: Ett objekt är en sak/del av programmet. Objektet har egenskaper och beteenden. 
Egenskaper och beteendet bestäms av klasser som fungerar som en mall för objektet. 
Exempel från verkligheten kan vara grundlagarna som skulle fungera som en klass och vi människor som följer denna mall skulle vara objektet. 

Konstruktor: Konstruktorn är en metod för att initiera objektets variabler. Den används för att sätta startvärden när objektet skapas.

Attribut: Ett attribut är en egenskap hos ett objekt. Attributet lagrar denna egenskap så att metoder i klassen kan använda informationen.

Versionshantering: Ett sätt för utvecklare att jobba i grupp med gemensamma projekt utan att förstöra koden genom branches och merges. Man kan se tidigare versioner och ändringar  av koden, samt vem som har gjort ändringarna.

TDD: “Test-Driven Development” innebär att man skriver test innan man skriver koden. 
1. Gör testet
2. Kör test (det kommer bli “fail” eftersom koden inte finns ännu
3. Skriv koden för testet
4. Kör testet igen och se så det blir “passed”
5. Finjustera koden utan att testerna får “fail”
